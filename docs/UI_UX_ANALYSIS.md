# WanMap UI/UX 包括分析レポート

**分析日**: 2025-12-12  
**対象バージョン**: 1.0.0 (9)  
**分析範囲**: 全画面・全機能

---

## 📊 分析サマリー

### ✅ 優れている点
1. **統一されたデザインシステム** - WanMapColors, Typography, Spacingで一貫性あり
2. **アイコンの統一** - ヘッダー/フッター間で最近統一完了
3. **ローディング状態** - Shimmer効果で適切なフィードバック
4. **ダークモード対応** - 全画面で完全対応

---

## 🔴 重要度：高（ユーザー体験に直結）

### 1. ホーム画面（HomeTab）
#### 問題点
- **地図プレビューが大きすぎる** (300px) - スクロール量が増えてコンテンツが見えにくい
- **人気No.1ルートのヘッダーが目立ちすぎ** - グラデーションが強すぎる
- **箱根エリアカードが巨大** (160px高) - 他エリアとのバランスが悪い

#### 推奨改善
1. 地図プレビュー高さを200pxに縮小
2. 人気No.1ヘッダーをシンプルなバッジスタイルに変更
3. 箱根エリアカードを120pxに縮小

---

### 2. ルート詳細画面（RouteDetailScreen）
#### 問題点
- **散歩開始ボタンの位置が低い** - ピンセクションの前に配置されスクロール必要
- **統計情報が横並びで詰まっている** - 距離・時間・難易度が読みにくい

#### 推奨改善
1. Floating Action Button (FAB)で散歩開始ボタンを固定
2. 統計情報を2行×2列のグリッドレイアウトに変更

---

### 3. マップページ（MapTab）
#### 問題点
- **現在地ボタンがない** - ユーザーが地図上で迷子になる
- **周辺ルートカードが小さすぎる** - タップしづらい（高さ不足）

#### 推奨改善
1. 右下に現在地ボタン追加（GPS再取得）
2. 周辺ルートカードの高さを100pxに拡大

---

### 4. プロフィール画面（ProfileTab）
#### 問題点
- **統計情報が2行表示で見づらい** - 「総散歩回数」「総距離」が縦並び
- **プロフィール編集への導線が弱い** - メニュー内に埋もれている

#### 推奨改善
1. 統計情報を3列グリッド（散歩回数・距離・時間）に変更
2. プロフィールカードに「編集」ボタンを直接配置

---

## 🟡 重要度：中（利便性向上）

### 5. ピン投稿カード（RecentPinCard）
#### 問題点
- **いいね・コメント・ブックマークボタンが小さい** (size: 16) - タップミス多発

#### 推奨改善
1. アクションボタンサイズを20pxに拡大 + タップ領域を48x48に

---

### 6. 人気ルートカード（PopularRouteCard）
#### 問題点
- **サムネイルが正方形80x80で小さい** - ルートの魅力が伝わりにくい
- **月間散歩回数の表示が小さい** - 「32回」が目立たない

#### 推奨改善
1. サムネイルを100x80の横長に変更
2. 月間散歩回数をバッジスタイルで強調表示

---

## 🎯 優先実装順位（DMOプレゼン前）

### Phase 1: 最優先（今すぐ実装）
1. **ホーム画面の地図高さ縮小** (300px → 200px)
2. **ルート詳細にFAB追加** (散歩開始ボタン)
3. **マップページに現在地ボタン追加**

### Phase 2: 高優先（プレゼン前に実装）
4. **統計情報のグリッドレイアウト化** (プロフィール・ルート詳細)
5. **ピン投稿カードのアクションボタン拡大** (16px → 20px)
6. **人気ルートカードのサムネイル拡大** (80x80 → 100x80)
